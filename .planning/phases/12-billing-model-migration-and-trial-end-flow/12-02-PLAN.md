  <name>Task 1: Trial notification dispatch logic</name>
  <files>
    src/lib/cron/trialNotification.ts
  </files>
  <action>
    Create src/lib/cron/trialNotification.ts with runTrialNotificationDispatch() function.

    Pattern: Follow the existing milestoneDispatch.ts pattern — service client, batch query, iterate, send, update flag.

    Steps inside runTrialNotificationDispatch():

    1. Query all subscriptions where status='trialing' AND trial_ends_at IS NOT NULL.
       Join to hotels table to get: hotel_id, hotels.name, hotels.owner_telegram_chat_id, hotels.country.
       Use (supabase as unknown as SupabaseClient) cast pattern.

    2. For each subscription, compute daysRemaining = Math.ceil((trial_ends_at - now) / (1000*60*60*24)).

       IMPORTANT: Use else-if chaining ordered most-recent-first so only ONE notification is sent per hotel per cron run. This prevents batch catch-up sends when a hotel is first checked after multiple thresholds have passed:

       if (daysRemaining <= 0 AND NOT trial_notified_day_14) => day 14 handler
       else if (daysRemaining <= 1 AND NOT trial_notified_day_13) => send day 13
       else if (daysRemaining <= 2 AND NOT trial_notified_day_12) => send day 12
       else if (daysRemaining <= 7 AND NOT trial_notified_day_7) => send day 7

    3. Skip hotels where owner_telegram_chat_id is NULL (not yet onboarded via Telegram).

    4. For each eligible notification, resolve the front desk bot token via:
       Query hotel_bots WHERE hotel_id = X AND role = 'front_desk' AND is_active = true.
       Then call supabase.rpc('get_bot_token', { p_vault_secret_id: bot.vault_secret_id }) to get the plaintext token.
       Use the front desk bot to send the notification (it is the "main" bot the owner interacts with).

    5. Send Telegram message using the sendTelegramReply helper from src/lib/telegram/sendReply.ts.
       Message content by notification day:
       - Day 7: "Your OtelAI trial has 7 days remaining. All your AI employees are working for you! Enjoy the rest of your trial."
       - Day 12 (2 days left): "Your OtelAI trial ends in 2 days. You'll be asked to select which AI employees to keep. Start thinking about which roles are most valuable to your hotel."
       - Day 13 (1 day left): "Your OtelAI trial ends tomorrow. After expiry, you'll select your AI team and complete payment to keep them active."
       - Day 14 (0 days left / expired): Instead of sending a plain text message, call sendTrialSelectionKeyboard from '@/lib/billing/trialSelection' (created in Plan 12-01).
         Pass { hotelId, chatId: owner_telegram_chat_id, botToken: <resolved front desk bot token>, currency: hotel.country === 'TR' ? 'try' : 'eur' }.
         This directly presents the employee selection inline keyboard to the owner, completing the trial-end flow trigger.

    6. After successful send, update the subscription row to set the corresponding trial_notified_day_X = TRUE.
       Use the SupabaseClient cast pattern for the update.

    7. Return { processed: number; sent: number; errors: number } for logging.

    Error handling: Wrap each hotel's notification in try/catch. Log errors, continue to next hotel. Return 200 even on partial failure (same pattern as milestoneDispatch and housekeeping cron).

    Import sendTelegramReply from '@/lib/telegram/sendReply'.
    Import createServiceClient from '@/lib/supabase/service'.
    Import { sendTrialSelectionKeyboard } from '@/lib/billing/trialSelection'.
