Build the trial-end employee selection flow: inline keyboard for toggling employees, payment link generation, and bot deactivation for unselected employees.

Output: Redis selection state, inline keyboard builder, trial callback handler, extended employee bot webhook to handle callback_query.

      Key pattern: trialSelect:{chatId}, 1-hour TTL
    - setTrialSelection(chatId: number, state: TrialSelection): Promise<void>

    - clearTrialSelection(chatId: number): Promise<void>

    Also export sendTrialSelectionKeyboard function:
    async function sendTrialSelectionKeyboard(params: {
      hotelId: string;
      chatId: number;
      botToken: string;
      currency: 'try' | 'eur';
    }): Promise<void>

    This function:
    1. Queries hotel_bots WHERE hotel_id AND is_active = true to get available roles.
    2. Initializes TrialSelection with all roles selected by default (owner starts with everything, deselects what they don't want).
    3. Builds the inline keyboard via buildSelectionKeyboard (from trialKeyboard.ts).
    4. Sends the message via Telegram Bot API sendMessage with the inline keyboard.
    5. Saves the returned message_id to TrialSelection state (needed for editMessageReplyMarkup).
    6. Stores the TrialSelection in Redis.

    Message text: "Your trial has ended. Select which AI employees to keep:\n\n[list of employees with prices]\n\nTotal: [amount] [currency]/month\n\nTap an employee to toggle. Tap Confirm to proceed to payment."

    export async function handleTrialCallback(
      callbackQuery: TelegramCallbackQuery,
      botRow: { hotel_id: string; vault_secret_id: string; role: string }
    ): Promise<void>

    Import TelegramCallbackQuery from types.ts. Import get/set/clearTrialSelection from trialSelection.ts. Import buildSelectionKeyboard from trialKeyboard.ts. Import EMPLOYEE_ROLE_PRICES, calculateMonthlyTotal, EmployeeRoleKey from plans.ts. Import getProviderForHotel from plans.ts. Import createServiceClient.

    Always call answerCallbackQuery FIRST (same pattern as wizard callback handler — dismiss loading spinner before async work).

    Handle callback_data patterns:

    1. "trial_toggle:{shortCode}" (e.g. "trial_toggle:fd"):
       - Get TrialSelection from Redis for this chatId
       - If no selection state, return silently (expired or not initiated)
       - Find the role key matching the shortCode via EMPLOYEE_ROLE_PRICES lookup
       - Toggle: if role is in selectedRoles, remove it; if not, add it
       - Must have at least 1 role selected — if removal would make selectedRoles empty, answerCallbackQuery with text "You must keep at least one employee" and return without toggling
       - Update Redis state with new selectedRoles
       - Build new keyboard with buildSelectionKeyboard
       - Build new message text showing updated selections and total
       - Call Telegram editMessageText API to update the existing message (use messageId from TrialSelection state) with new text AND new reply_markup (keyboard)

    2. "trial_all":
       - Get TrialSelection from Redis
       - Set selectedRoles = availableRoles (select all)
       - Update Redis state
       - editMessageText with updated keyboard and total

    3. "trial_confirm":
       - Get TrialSelection from Redis
       - If no state, return silently
       - If selectedRoles is empty, answerCallbackQuery with "Select at least one employee" and return
       - Calculate total using calculateMonthlyTotal(selectedRoles, currency)
       - Deactivate unselected bots: for each role in availableRoles that is NOT in selectedRoles:
         UPDATE hotel_bots SET is_active = false WHERE hotel_id = X AND role = Y
         Use service client with SupabaseClient cast
       - Generate payment link based on provider:
         - Query hotels table for country to determine provider via getProviderForHotel(country)
         - For Mollie (EU): use mollieClient.paymentLinks.create() from @mollie/api-client
           Amount: { currency: 'EUR', value: total.toFixed(2) }
           Description: "OtelAI - {N} AI employees"
           redirectUrl: process.env.NEXT_PUBLIC_APP_URL + '/billing?status=success'
           Set metadata: { hotelId, selectedRoles: selectedRoles.join(',') }
           Extract paymentLink._links.paymentLink.href as the link URL
         - For iyzico (TR): generate a link to the web dashboard billing page instead:
           paymentUrl = process.env.NEXT_PUBLIC_APP_URL + '/billing?action=subscribe&roles=' + selectedRoles.join(',') + '&total=' + total
           (Per research: iyzico Checkout Form requires buyer object with Turkish national ID — cannot generate a direct payment link. Send owner to web dashboard billing page which collects buyer info and calls iyzico checkout API)
       - Send Telegram message to chatId with the payment link:
         "Your selection: {selectedRoles as display names}\nMonthly total: {total} {currency}\n\nComplete payment here: {paymentUrl}\n\nUnselected employees have been deactivated. They will reactivate after payment is confirmed."
       - Clear Redis trial selection state
       - editMessageText on the original selection message to show "Selection confirmed. Proceed to payment." (removes the inline keyboard)

    Error handling: Wrap each handler section in try/catch. Log errors, never throw. answerCallbackQuery should be called even on error paths.

    For Mollie payment link: import { mollieClient } from '@/lib/billing/mollie'. The Mollie Payment Links API creates a reusable payment link. Check if mollieClient.paymentLinks exists; if not, fall back to creating a one-time payment via mollieClient.payments.create with redirectUrl.
