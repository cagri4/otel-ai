1. Create `src/lib/security/sanitizeGuestInput.ts`:
   - Export `sanitizeGuestInput(input: string): string`
   - Length cap: 2000 characters via `.slice(0, 2000)`
   - Unicode normalization: `.normalize('NFC')`
   - Remove null bytes and control characters (keep newlines/tabs): `.replace(/[\x00-\x08\x0B\x0C\x0E-\x1F\x7F]/g, '')`
   - Check against INJECTION_PATTERNS array (per research Pattern 5):
     - `/ignore\s+(all\s+)?(previous|prior|above)\s+instructions?/i`
     - `/you\s+are\s+now\s+(in\s+)?developer\s+mode/i`
     - `/forget\s+(everything|all|your)\s+(you\s+)?(know|were\s+told)/i`
     - `/\[SYSTEM\]/i`
     - `/\[INST\]/i`
     - `/<\|system\|>/i`
     - `/act\s+as\s+if\s+you\s+(have\s+no|are\s+not)/i`
     - `/jailbreak/i`
   - If any pattern matches, return `'[Message could not be processed. Please rephrase your question.]'`
   - Return `.trim()`

2. Create `src/lib/security/rateLimiter.ts`:
   - Import `Ratelimit` from `@upstash/ratelimit` and `Redis` from `@upstash/redis`
   - Create a `getRedis()` function that lazily initializes Redis from env vars (UPSTASH_REDIS_REST_URL, UPSTASH_REDIS_REST_TOKEN)
   - If env vars missing, return null (graceful degradation)
   - Export `ipRateLimiter`: sliding window, 30 requests per 60 seconds, prefix `rl:ip`
   - Export `hotelRateLimiter`: fixed window, 100 requests per 60 seconds, prefix `rl:hotel`
   - Both return `null` if Redis is unavailable (rate limiting should not block if Redis is down)
   - Export `checkIpRateLimit(ip: string): Promise<{ success: boolean }>` — calls ipRateLimiter.limit(ip), returns { success: true } if limiter is null
   - Export `checkHotelRateLimit(hotelId: string): Promise<{ success: boolean }>` — calls hotelRateLimiter.limit(hotelId), returns { success: true } if limiter is null

3. Update `src/lib/supabase/middleware.ts`:
   - Add public route bypass BEFORE the Supabase auth check. Routes starting with `/api/widget`, `/api/whatsapp`, `/widget`, or `/api/escalations` should skip auth entirely (return `NextResponse.next()` with no session refresh).
   - This is needed because widget guests and WhatsApp webhooks have no Supabase session.
   - Keep the existing auth logic for all other routes unchanged.

4. Update `src/middleware.ts`:
   - Import `checkIpRateLimit` from `@/lib/security/rateLimiter`
   - For requests matching `/api/widget/*` or `/api/whatsapp/*`:
     - Extract IP via `request.headers.get('x-forwarded-for')?.split(',')[0]?.trim() ?? request.ip ?? '127.0.0.1'`
     - Call `checkIpRateLimit(ip)`
     - If `!success`, return `new NextResponse('Too Many Requests', { status: 429, headers: { 'Retry-After': '60' } })`
   - For all other requests, continue to `updateSession(request)` as before
   - Keep the existing matcher but ensure it covers widget and whatsapp routes

IMPORTANT: The middleware must NOT use Edge-only imports. `@upstash/ratelimit` and `@upstash/redis` are HTTP-based and work in both Node.js and Edge runtimes.
