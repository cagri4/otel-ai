<objective>
Build the escalation notification system: detect when the Front Desk AI cannot handle a guest request (by checking for fallback phrases in the response), insert an escalation record into the database, and send an email notification to the hotel owner via Resend. The detection runs as a fire-and-forget side effect after the agent responds — it never blocks the guest-facing response flow.

Purpose: Hotel owners need to know within 2 minutes when a guest has a request the AI cannot handle (DESK-06). This closes the loop between AI automation and human intervention, ensuring no guest request goes unnoticed.

Output: Escalation detection function, escalation API endpoint, updated invokeAgent with escalation hook.
</objective>

1. Create `src/lib/agents/escalation.ts`:

   Define `ESCALATION_PHRASES` constant array:
   ```
   'please contact reception',
   'please call us directly',
   'i cannot help with this',
   'outside my capabilities',
   'please speak with a staff member',
   'i am unable to assist',
   'i recommend contacting the hotel directly',
   'this is beyond what i can do',
   ```

   Export `async function detectAndInsertEscalation(params: { response: string; userMessage: string; conversationId: string; hotelId: string; channel: 'whatsapp' | 'widget' | 'dashboard' }): Promise<void>`:
   - Determine channel from conversationId prefix: `wa_` = 'whatsapp', `widget_` = 'widget', else 'dashboard'
   - Check if `response.toLowerCase()` contains any ESCALATION_PHRASES
   - If NO match, return early (no escalation needed)
   - If match found:
     - Import `createServiceClient` from `@/lib/supabase/service`
     - Insert into `escalations` table: `{ hotel_id, conversation_id, channel, guest_message: userMessage, agent_response: response }`
     - Fire the notification: POST to `${process.env.NEXT_PUBLIC_APP_URL}/api/escalations` with the escalation record as body
       - Use `fetch()` with a 5-second timeout via AbortSignal.timeout(5000)
       - Do NOT await the full response — fire-and-forget is acceptable
     - Wrap everything in try/catch — log errors but never throw (this is a side effect, must not crash the agent)

2. Create `src/app/api/escalations/route.ts`:
   ```
   export const runtime = 'nodejs';
   export const dynamic = 'force-dynamic';
   ```

   Export `async function POST(req: Request): Promise<Response>`:
   - Parse JSON body: `{ hotel_id, conversation_id, channel, guest_message, agent_response }`
   - Import `Resend` from `resend`
   - Create Resend client: `new Resend(process.env.RESEND_API_KEY)`
   - Query hotel's contact_email using `createServiceClient()`:
     ```
     SELECT name, contact_email FROM hotels WHERE id = hotel_id
     ```
   - If no contact_email, log warning and return 200 (cannot notify)
   - Send email via Resend:
     ```
     await resend.emails.send({
       from: process.env.RESEND_FROM_EMAIL ?? 'noreply@otelai.com',
       to: hotel.contact_email,
       subject: `[${hotel.name}] Guest needs assistance`,
       html: `
         <h2>Guest Escalation Alert</h2>
         <p><strong>Channel:</strong> ${channel}</p>
         <p><strong>Guest said:</strong></p>
         <blockquote>${guest_message}</blockquote>
         <p><strong>AI responded:</strong></p>
         <blockquote>${agent_response}</blockquote>
         <p><strong>Conversation ID:</strong> ${conversation_id}</p>
         <p>Please follow up with your guest as soon as possible.</p>
       `,
     });
     ```
   - Update escalation record: SET `notified_at = NOW()` WHERE `id = record.id` (if id provided) or WHERE `conversation_id = conversation_id AND notified_at IS NULL`
   - Return `{ ok: true }`
   - Wrap in try/catch; return 500 on error

NOTE: Use plain HTML for the email template — do NOT use React Email components. React Email is not installed and adds unnecessary complexity for a single notification email. Plain HTML is sufficient for Phase 4.
