1. Create `src/lib/supabase/service.ts`:
   - Import `createClient` from `@supabase/supabase-js`
   - Export `function createServiceClient()`:
     - Returns `createClient(process.env.NEXT_PUBLIC_SUPABASE_URL!, process.env.SUPABASE_SERVICE_ROLE_KEY!)`
     - This client bypasses RLS — used server-side only for widget/WhatsApp operations where there is no user session
   - Add a JSDoc warning: "NEVER expose this client to browser code. Server-only."

2. Create `src/app/api/widget/session/route.ts`:
   ```
   export const runtime = 'nodejs';
   export const dynamic = 'force-dynamic';
   ```
   Export `async function POST(req: Request): Promise<Response>`:
   - Parse JSON body: `{ token: string }`
   - Validate token is a non-empty string; return 400 if missing
   - Use `createServiceClient()` to query `hotels` table WHERE `widget_token = token`
   - Select: `id, name, widget_config`
   - If not found, return 404 `{ error: 'Invalid widget token' }`
   - Generate `conversationId`: `widget_${hotel.id}_${crypto.randomUUID()}`
   - Return JSON:
     ```json
     {
       "conversationId": "widget_...",
       "hotelId": "uuid",
       "hotelName": "Hotel Name",
       "widgetConfig": { ... },
       "channel": "widget_responses:widget_..."
     }
     ```

3. Create `src/app/api/widget/message/route.ts`:
   ```
   export const runtime = 'nodejs';
   export const dynamic = 'force-dynamic';
   ```
   Export `async function POST(req: Request): Promise<Response>`:
   - Parse JSON body: `{ message: string, conversationId: string }` (NOTE: hotelId is NOT accepted from client — it is parsed server-side)
   - Validate both fields; return 400 if missing
   - **Parse hotelId server-side from conversationId**: Extract from the `widget_{hotelId}_{uuid}` format:
     ```typescript
     const parts = conversationId.split('_');
     if (parts.length < 3 || parts[0] !== 'widget') {
       return Response.json({ error: 'Invalid conversationId' }, { status: 400 });
     }
     const hotelId = parts[1];
     ```
   - This prevents a malicious client from sending a different hotelId than the one the conversation was created for.
   - Call `checkHotelRateLimit(hotelId)`; return 429 if rate limited
   - Call `sanitizeGuestInput(message)`
   - Call `invokeAgent({ role: AgentRole.FRONT_DESK, userMessage: sanitized, conversationId, hotelId })` — non-streaming
   - Broadcast response via Supabase Realtime:
     ```
     const supabase = createServiceClient();
     const channel = supabase.channel(`widget_responses:${conversationId}`);
     await channel.send({
       type: 'broadcast',
       event: 'message',
       payload: { role: 'assistant', content: response, created_at: new Date().toISOString() },
     });
     await supabase.removeChannel(channel);
     ```
   - Return `{ ok: true }`
   - Wrap in try/catch; on error return 500 `{ error: 'Failed to process message' }`
  </action>

   **State:**
   - `messages: Array<{ role: 'user' | 'assistant'; content: string; id: string }>`
   - `input: string`
   - `isLoading: boolean`
   - `conversationId: string | null`
   - `hotelId: string | null`
   - `error: string | null`

   **Init (useEffect on mount):**
   - Accept `token` prop
   - POST to `/api/widget/session` with `{ token }`
   - On success: set conversationId, hotelId from response
   - Subscribe to Supabase Realtime Broadcast channel `widget_responses:${conversationId}`:
     ```
     const supabase = createClient(NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY);
     const channel = supabase.channel(channelName)
       .on('broadcast', { event: 'message' }, ({ payload }) => {
         setMessages(prev => [...prev, { ...payload, id: crypto.randomUUID() }]);
         setIsLoading(false);
       })
       .subscribe();
     ```
   - Cleanup: `supabase.removeChannel(channel)` on unmount

   **Send message handler:**
   - Add user message to `messages` optimistically
   - Set `isLoading = true`
   - POST to `/api/widget/message` with `{ message: input, conversationId }` (hotelId is NOT sent — server parses it from conversationId)
   - Clear input
   - On error: set error state, set isLoading false

   **UI (Tailwind):**

   - Header: hotel name (from session response), styled with widget primary color if available
   - Welcome message from widgetConfig shown when messages array is empty

   - Input area: text input + send button at bottom; disabled when loading
   - Loading indicator: pulsing dots in an assistant bubble while waiting for response
   - Error display: subtle red text above input
