2. **Create `src/lib/telegram/wizard/wizardState.ts`:**
   - Define `WizardStep` union type: `'collect_hotel_name' | 'collect_address' | 'collect_room_count' | 'collect_checkin_time' | 'collect_checkout_time' | 'confirm_complete'`
   - Define `WizardState` interface: `{ hotelId: string, step: WizardStep, hotelName?: string, address?: string, roomCount?: number, checkinTime?: string, checkoutTime?: string }`
   - Create `getRedis()` helper following the lazy-init pattern from `rateLimiter.ts` (with `let _redis: Redis | null | undefined = undefined` pattern)
   - Export `getWizardState(chatId: number): Promise<WizardState | null>` — reads `wizard:{chatId}` key from Redis. Returns `null` if Redis unavailable OR key not found.
   - Export `setWizardState(chatId: number, state: WizardState): Promise<void>` — writes to `wizard:{chatId}` with `{ ex: 604800 }` (7-day TTL). Resets TTL on every write so active sessions never expire mid-wizard.
   - Export `clearWizardState(chatId: number): Promise<void>` — deletes `wizard:{chatId}` on completion.
   - Export `isRedisAvailable(): boolean` — returns whether getRedis() produces a non-null client. Used by the webhook handler to distinguish "Redis down" from "no session found".
   - Use `@upstash/redis` import (already installed, same as rateLimiter.ts)
  </action>

<task type="auto">
  <name>Task 2: Create wizard step machine, message helper, and completion logic</name>
  <files>src/lib/telegram/wizard/wizardSteps.ts, src/lib/telegram/wizard/wizardActions.ts</files>
  <action>
1. **Create `src/lib/telegram/wizard/wizardSteps.ts`:**
   - Import `setWizardState`, `WizardState` from `./wizardState`
   - Import `createServiceClient` from `@/lib/supabase/service`
   - Import `sanitizeGuestInput` from `@/lib/security/sanitizeGuestInput`
   - Import `SupabaseClient` type from `@supabase/supabase-js` (for the cast pattern)
   - Export `advanceWizard(chatId: number, state: WizardState, userInput: string, botToken: string): Promise<void>`
   - Implement as a switch/case on `state.step`:
     - `collect_hotel_name`: sanitize input, update `hotels.name` where `id = state.hotelId` via `(supabase as unknown as SupabaseClient).from('hotels').update({name}).eq('id', state.hotelId)`, set state to `collect_address`, send "What is your hotel address?"
     - `collect_address`: sanitize, update `hotels.address`, set state to `collect_room_count`, send "How many rooms does your hotel have?"
     - `collect_room_count`: parse as integer, validate > 0 (if invalid, send "Please enter a valid number of rooms (e.g. 20)" and return without advancing), upsert hotel_fact with category 'policy' and content `The hotel has ${roomCount} rooms.`, set state to `collect_checkin_time`, send "What time is check-in? (e.g. 3 PM or 15:00)"
     - `collect_checkin_time`: sanitize, upsert hotel_fact with category 'policy' and content `Check-in time is ${sanitized}.`, set state to `collect_checkout_time`, send "What time is check-out? (e.g. 11 AM or 11:00)"
     - `collect_checkout_time`: sanitize, upsert hotel_fact with category 'policy' and content `Check-out time is ${sanitized}.`, set state to `confirm_complete`, send confirmation summary with inline keyboard
   - For the `confirm_complete` step transition at checkout_time: build a summary string showing hotelName, address, roomCount, checkinTime, checkoutTime. Call `sendWizardMessage` with inline keyboard reply_markup: `{ inline_keyboard: [[{ text: "Yes, activate!", callback_data: "wizard:confirm" }, { text: "Start over", callback_data: "wizard:restart" }]] }`
   - For `upsertHotelFact`: use `(supabase as unknown as SupabaseClient).from('hotel_facts').insert({ hotel_id: state.hotelId, category, content }).select()`. If insert fails (duplicate), that is acceptable — use upsert or just insert.
   - Import and use `sendWizardMessage` from `./wizardActions` for all outbound messages

2. **Create `src/lib/telegram/wizard/wizardActions.ts`:**
   - Import `getWizardState`, `setWizardState`, `clearWizardState`, `isRedisAvailable` from `./wizardState`
   - Import `escapeMarkdownV2` from `@/lib/telegram/escapeMarkdownV2`
   - Import `sanitizeGuestInput` from `@/lib/security/sanitizeGuestInput`
   - Import `createServiceClient` from `@/lib/supabase/service`
   - Import `SupabaseClient` type from `@supabase/supabase-js`
   - Import `advanceWizard` from `./wizardSteps`

   **Export `sendWizardMessage(botToken, chatId, text, replyMarkup?)`: Promise<void>:**
   - Same MarkdownV2 + plaintext fallback pattern as `sendTelegramReply` in sendReply.ts
   - Accept optional `replyMarkup` object for inline keyboard
   - Build body with `chat_id`, `text: escapeMarkdownV2(text)`, `parse_mode: 'MarkdownV2'`, and optionally `reply_markup`
   - On non-ok response, fallback to plaintext (no parse_mode, original text)
   - Never throw — catch and console.error

   **Export `handleWizardMessage(message: TelegramMessage)`: Promise<void>:**
   - Extract `chatId = message.chat.id`, `text = message.text ?? ''`
   - Read `botToken = process.env.SETUP_WIZARD_BOT_TOKEN ?? ''`
   - Check for `/start` with payload: `const match = text.match(/^\/start\s+([A-Za-z0-9_-]{1,64})$/)`
   - If `/start {payload}`:
     - Validate payload is a UUID format (36 chars with hyphens)
     - Check if an existing wizard session exists for this chatId. If it does and `state.hotelId === payload`, resume: send "You have an active setup session. Pick up where you left off:" + ask the current step's question. Return.
     - Query `hotels` table via service client to verify hotel exists. If not found, send "Invalid setup link. Please contact your administrator." and return.
     - Initialize wizard state: `{ hotelId: payload, step: 'collect_hotel_name' }`
     - If `hotel.name` exists and is not 'My Hotel', pre-fill `hotelName` in state
     - Send welcome: "Welcome! I am your OtelAI setup assistant.\n\nLet's start with your hotel name. What is the full name of your hotel?"
     - Return
   - If no `/start`:
     - Check `isRedisAvailable()`. If false, send "Setup service is temporarily unavailable. Please try again later." and return.
     - Get wizard state for chatId. If null, send "No active setup session. Please use the setup link provided by your administrator." and return.
     - If state.step is `confirm_complete`, send "Please use the buttons above to confirm or restart." and return (don't process text input on confirmation step).
     - Call `advanceWizard(chatId, state, text, botToken)`

   **Export `handleWizardCallback(callbackQuery: TelegramCallbackQuery)`: Promise<void>:**
   - Extract `chatId = callbackQuery.message?.chat.id`, `callbackId = callbackQuery.id`, `data = callbackQuery.data ?? ''`
   - If no chatId, return
   - Read `botToken = process.env.SETUP_WIZARD_BOT_TOKEN ?? ''`
   - Call `answerCallbackQuery` FIRST: fetch `https://api.telegram.org/bot${botToken}/answerCallbackQuery` with `{ callback_query_id: callbackId }` — dismiss loading spinner
   - If `data === 'wizard:confirm'`:
     - Get wizard state. If null or step !== 'confirm_complete', return.
     - Call `completeWizard(chatId, state, botToken)` (private function)
   - If `data === 'wizard:restart'`:
     - Get wizard state. If null, return.
     - Reset state: `{ hotelId: state.hotelId, step: 'collect_hotel_name' }`
     - Send "No problem! Let's start again.\n\nWhat is the full name of your hotel?"

   **Private `completeWizard(chatId, state, botToken)`: Promise<void>:**
   - Create service client
   - Write `hotels.onboarding_completed_at = new Date().toISOString()` where `id = state.hotelId` — using the `(supabase as unknown as SupabaseClient)` cast pattern
   - Do NOT touch the `subscriptions` table — trial was already created by `seed_hotel_defaults` at hotel creation (Phase 10). Per research Pitfall 4.
   - Fetch `hotel_bots` rows: `(supabase as unknown as SupabaseClient).from('hotel_bots').select('role, bot_username').eq('hotel_id', state.hotelId).eq('is_active', true)`
   - Build role labels: `{ front_desk: 'Front Desk', booking_ai: 'Booking AI', guest_experience: 'Guest Experience', housekeeping_coordinator: 'Housekeeping' }`
   - Build bot links. If bots found, format as bullet list with `t.me/{bot_username}` links. If no bots found, add message: "Your administrator will activate your employee bots shortly."
   - Build completion message: "Setup complete! Your hotel AI team is ready.\n\nYour employee bots:\n{botLinks}\n\nYour 14-day trial has started. Enjoy!"
   - Send via `sendWizardMessage`
   - Clear wizard state: `clearWizardState(chatId)`
