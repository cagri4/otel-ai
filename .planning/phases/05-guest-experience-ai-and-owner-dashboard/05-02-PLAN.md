This fires daily at 06:00 UTC. The milestoneDispatch logic uses per-hotel timezone for date comparisons, so one UTC cron handles all timezones correctly.

**2. Create `src/app/api/cron/milestone-dispatch/route.ts`**:
```typescript
import type { NextRequest } from 'next/server';
import { runMilestoneDispatch } from '@/lib/cron/milestoneDispatch';

export const runtime = 'nodejs';
export const dynamic = 'force-dynamic';
export const maxDuration = 300; // Pro plan: up to 300s for batch processing

export async function GET(request: NextRequest) {
  // Verify CRON_SECRET — Vercel sets Authorization header on cron invocations
  const authHeader = request.headers.get('authorization');
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return new Response('Unauthorized', { status: 401 });
  }

  try {
    const result = await runMilestoneDispatch();
    return Response.json({ ok: true, ...result });
  } catch (error) {
    const message = error instanceof Error ? error.message : String(error);
    console.error('[cron/milestone-dispatch] Fatal error:', message);
    // Return 200 even on error — Vercel cron does not retry on 5xx (only single attempt),
    // but returning 200 keeps the logs clean. Errors are logged for debugging.
    return Response.json({ ok: false, error: message }, { status: 200 });
  }
}
```

Create `src/lib/cron/milestoneDispatch.ts` with the following structure:

**1. Main function `runMilestoneDispatch()`:**
- Uses service-role Supabase client (no user session in cron context).
- Fetches all hotels with their timezone and name: `supabase.from('hotels').select('id, timezone, name, contact_email')`.
- For each hotel, computes today/tomorrow/yesterday in the hotel's timezone using `formatInTimeZone` from `date-fns-tz` and `addDays` from `date-fns`.
- Queries bookings for each milestone:
  - Pre-arrival: `check_in_date = tomorrowStr AND pre_arrival_sent = false`
  - Checkout reminder: `check_out_date = todayStr AND checkout_reminder_sent = false`
  - Post-stay review: `check_out_date = yesterdayStr AND review_request_sent = false`
- Also checks `agents` table for `guest_experience` role `is_enabled` per hotel — skip hotel if disabled.
- Loads message templates for the hotel: `supabase.from('message_templates').select('*').eq('hotel_id', hotel.id)`.
- For each booking in each milestone group, dispatches the message and marks the sent flag.

**2. Message dispatch per milestone:**
For each booking, determine the channel (booking.channel field):

- **WhatsApp channel**: Use Twilio. For pre-arrival and checkout reminder, attempt free-form body first (guest may be within 24h window). For post-stay review requests (always outside 24h window), use `contentSid` + `contentVariables` if `TWILIO_TEMPLATE_SID_REVIEW_REQUEST` env var is set. If template env var is not set, fall back to email for post-stay reviews.

  Create a helper `sendMilestoneWhatsApp(params)` that wraps Twilio `client.messages.create()`:
  ```typescript
  const client = twilio(process.env.TWILIO_ACCOUNT_SID!, process.env.TWILIO_AUTH_TOKEN!);
  // Free-form for in-window milestones
  await client.messages.create({
    from: `whatsapp:${process.env.TWILIO_WHATSAPP_NUMBER}`,
    to: booking.guest_phone!, // Already whatsapp:+xxx formatted
    body: messageBody,
  });
  ```

- **Email channel**: Use Resend. Create a helper `sendMilestoneEmail(params)` wrapping `resend.emails.send()`:

  ```typescript
  await resend.emails.send({
    from: process.env.RESEND_FROM_EMAIL!,
    to: booking.guest_email!,
    subject: template?.subject || defaultSubject,
    html: messageBody, // Simple HTML wrapping the text
  });
  ```

**3. Message content generation:**
For each milestone, check if a custom template exists for this hotel + milestone + channel combination:
- If template found: use template.body, substituting `{{guest_name}}`, `{{hotel_name}}`, `{{check_in_date}}`, `{{check_out_date}}` placeholders with actual values.
- If no template: use built-in default messages:
  - Pre-arrival default: "Dear {guest_name}, we're looking forward to welcoming you to {hotel_name} tomorrow! Check-in starts at 3:00 PM. If you need anything before your arrival, don't hesitate to reach out."
  - Checkout default: "Good morning {guest_name}! Just a friendly reminder that checkout at {hotel_name} is at 11:00 AM today. We hope you've had a wonderful stay!"
  - Review default: "Dear {guest_name}, thank you for staying with us at {hotel_name}! We hope you enjoyed your visit. We would love to hear about your experience."

**4. After successful dispatch, update the booking's sent flag:**

Same pattern for checkout_reminder_sent and review_request_sent.

- Return a summary object: `{ hotelsProcessed, messagesSent, errors }`.
