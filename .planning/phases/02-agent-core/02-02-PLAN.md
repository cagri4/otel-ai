**Step 1: Create tool executor** (`src/lib/agents/tools/executor.ts`):

Create a dispatch function that maps tool names to implementations:

```typescript
import { getAvailability, getRoomPricing, lookupGuestReservation } from './stubs';

const TOOL_DISPATCH: Record<string, (input: Record<string, unknown>) => Promise<Record<string, unknown>>> = {
  get_room_availability: getAvailability,
  get_room_pricing: getRoomPricing,
  lookup_guest_reservation: lookupGuestReservation,
};

export async function executeTool(name: string, input: Record<string, unknown>): Promise<string> {
  const handler = TOOL_DISPATCH[name];
  if (!handler) throw new Error(`Unknown tool: ${name}`);
  const result = await handler(input);
  return JSON.stringify(result);
}
```

**Step 2: Create invokeAgent()** (`src/lib/agents/invokeAgent.ts`):

Import:
- `Anthropic` from `@anthropic-ai/sdk`
- `agentFactory` from `./agentFactory`
- `assembleSystemPrompt` from `./assembleContext`
- `loadConversationTurns`, `persistTurn` from `./memory`
- `executeTool` from `./tools/executor`
- `InvokeAgentParams` from `./types`
- `getToolsForRole` from `./tools/registry`

Create a module-level Anthropic client:
```typescript
const client = new Anthropic(); // Uses ANTHROPIC_API_KEY env var automatically
```

Implement `invokeAgent(params: InvokeAgentParams): Promise<string>`:

1. Get role config: `const config = agentFactory.getConfig(params.role)`
2. Assemble system prompt: `const systemPrompt = await assembleSystemPrompt({ role: params.role, hotelId: params.hotelId, conversationId: params.conversationId, config })`
3. Load working memory: `const messages = await loadConversationTurns(params.conversationId)`
4. Persist user turn: `await persistTurn(params.conversationId, params.hotelId, "user", params.userMessage)`
5. Append user message to messages array: `messages.push({ role: "user", content: params.userMessage })`
6. Determine tool_choice: Use `isToolRequired(params.userMessage)` helper that checks if the message contains keywords related to availability, pricing, rooms, booking, reservation, cost, rate. If yes: `{ type: "any" as const }`. If no: `{ type: "auto" as const }`. Only use "any" if the role's tools array is non-empty (per research pitfall 5).
7. Call Claude with streaming:
   ```typescript
   const stream = client.messages.stream({
     model: config.model,
     max_tokens: 4096,
     system: systemPrompt,
     tools: getToolsForRole(params.role),
     tool_choice: toolChoice,
     messages,
   });
   ```
8. Listen for text tokens and forward via `onToken` callback:
   ```typescript
   stream.on("text", (text) => {
     params.onToken?.(text);
   });
   ```
9. Get final message: `const finalMessage = await stream.finalMessage()`
10. Handle tool_use stop_reason:
    - If `finalMessage.stop_reason === "tool_use"`:
      a. Extract tool_use blocks from `finalMessage.content`
      b. For each tool_use block, call `executeTool(block.name, block.input as Record<string, unknown>)`
      c. Persist the assistant turn (as JSON stringified content blocks) with role "assistant"
      d. Build tool_result content blocks
      e. Persist each tool_result with role "tool" and the corresponding tool_use_id
      f. Recursively call Claude with updated messages (original messages + assistant content + tool results as user message)
      g. Apply a recursion depth limit of 5 to prevent infinite tool loops
      h. On the recursive call, also stream tokens via onToken callback
    - If `finalMessage.stop_reason === "end_turn"`:
      a. Collect full text from `finalMessage.content` (filter text blocks)
      b. Persist assistant turn: `await persistTurn(params.conversationId, params.hotelId, "assistant", fullText)`
      c. Return fullText

11. Create helper `isToolRequired(message: string): boolean`:
    - Lowercase the message
    - Check for keywords: "available", "availability", "price", "pricing", "cost", "rate", "room", "book", "booking", "reservation", "how much", "per night", "vacant", "free room"
    - Return true if any match
    - This is a simple keyword heuristic -- it errs on the side of requiring tools (false positives are acceptable; false negatives mean Claude might answer from training data)

**CRITICAL anti-patterns to avoid (from research):**
- Do NOT cache hotel data in module-level variables
- Do NOT call Claude from Server Components or Server Actions -- invokeAgent is called from Route Handlers only
- Do NOT use `await` in `ReadableStream.start()` when this is later wired to SSE (that's Plan 04's concern)
- Do NOT store conversation turns as plain text if they contain tool blocks -- store tool_use content as JSON string

**Error handling:**
- Wrap the Claude API call in try/catch
- If Anthropic API returns an error, throw with a descriptive message including the error type
- If tool execution fails, return an error result to Claude (don't crash): `{ error: true, message: "Tool execution failed: ..." }`
