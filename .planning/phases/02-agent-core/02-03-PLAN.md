  <name>Task 1: Agent coordination helpers for async task delegation</name>
  <files>src/lib/agents/coordination.ts</files>
  <action>
Create `src/lib/agents/coordination.ts` with functions that manage the agent_tasks lifecycle. Use the server Supabase client (`createClient` from `@/lib/supabase/server`). All queries go through RLS.

Functions to implement:

1. `delegateTask(params: { hotelId: string; fromRole: string; toRole: string; taskType: string; payload: Record<string, unknown> }): Promise<AgentTask>`
   - INSERT into agent_tasks with status 'pending'
   - Returns the created task row
   - Use case: Front Desk AI delegates "check_housekeeping_status" to Housekeeping role

2. `getPendingTasks(hotelId: string, forRole: string): Promise<AgentTask[]>`
   - SELECT from agent_tasks WHERE hotel_id = $1 AND to_role = $2 AND status = 'pending'
   - ORDER BY created_at ASC (FIFO)
   - Returns array of pending tasks

3. `claimTask(taskId: string): Promise<AgentTask>`
   - UPDATE agent_tasks SET status = 'processing', updated_at = NOW() WHERE id = $1 AND status = 'pending'
   - Returns the updated task
   - The WHERE status = 'pending' prevents double-claim race conditions

4. `completeTask(taskId: string, result: Record<string, unknown>): Promise<AgentTask>`
   - UPDATE agent_tasks SET status = 'completed', result = $2, completed_at = NOW(), updated_at = NOW() WHERE id = $1
   - Returns the updated task

5. `failTask(taskId: string, errorMessage: string): Promise<AgentTask>`
   - UPDATE agent_tasks SET status = 'failed', error_message = $2, updated_at = NOW() WHERE id = $1
   - Returns the updated task

All functions:
- Are async
- Use proper TypeScript types from `@/types/database` (AgentTask type)
- Throw descriptive errors on Supabase failures
- Use the server client (works from Route Handlers)
- Do NOT import or call invokeAgent() -- this module has NO dependency on the agent orchestrator (prevents circular deps)

Add a JSDoc comment at the top of the file explaining the fire-and-forget pattern:
```
/**
 * Agent-to-agent coordination via async tasks table.
 *
 * Pattern: Fire-and-forget delegation. An agent INSERTs a task row and
 * returns to the caller immediately. The delegated agent picks up the
 * task on its next invocation (or via a future cron/webhook trigger).
 *
 * Anti-pattern: NEVER call invokeAgent() from within this module or
 * from a tool handler. Always delegate via delegateTask() instead.
 */
```
