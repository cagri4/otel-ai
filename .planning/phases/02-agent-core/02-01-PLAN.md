  <action>
Create SQL migration `supabase/migrations/0002_agent_core.sql` containing:

1. **hotel_facts** table (semantic memory):
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `hotel_id UUID NOT NULL REFERENCES public.hotels(id) ON DELETE CASCADE`
   - `category TEXT NOT NULL` -- values: 'policy', 'amenity', 'faq', 'pricing_note'
   - `fact TEXT NOT NULL`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`
   - `updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`
   - Index: `idx_hotel_facts_hotel_id ON hotel_facts(hotel_id)`
   - RLS: enable, SELECT/INSERT/UPDATE/DELETE for authenticated users WHERE `hotel_id = ((SELECT auth.jwt()) ->> 'hotel_id')::uuid`

2. **guest_interactions** table (episodic memory):
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `hotel_id UUID NOT NULL REFERENCES public.hotels(id) ON DELETE CASCADE`
   - `guest_identifier TEXT NOT NULL` -- email, phone, or session token
   - `summary TEXT NOT NULL` -- agent-written summary
   - `sentiment TEXT` -- 'positive', 'neutral', 'negative'
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`
   - Index: `idx_guest_interactions_hotel_id ON guest_interactions(hotel_id, created_at)`
   - RLS: enable, SELECT/INSERT for authenticated users WHERE hotel_id matches JWT

3. **conversation_turns** table (working memory):
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `conversation_id UUID NOT NULL`
   - `hotel_id UUID NOT NULL REFERENCES public.hotels(id) ON DELETE CASCADE`
   - `role TEXT NOT NULL CHECK (role IN ('user', 'assistant', 'tool'))`
   - `content TEXT NOT NULL`
   - `tool_use_id TEXT` -- for tool/tool_result correlation
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`
   - Index: `idx_conversation_turns_conv ON conversation_turns(conversation_id, created_at)`
   - RLS: enable, SELECT/INSERT for authenticated users WHERE hotel_id matches JWT

4. **agent_tasks** table (coordination):
   - Create enum type: `CREATE TYPE public.task_status AS ENUM ('pending', 'processing', 'completed', 'failed')`
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `hotel_id UUID NOT NULL REFERENCES public.hotels(id) ON DELETE CASCADE`
   - `from_role TEXT NOT NULL`
   - `to_role TEXT NOT NULL`
   - `task_type TEXT NOT NULL`
   - `payload JSONB NOT NULL DEFAULT '{}'`
   - `status public.task_status NOT NULL DEFAULT 'pending'`
   - `result JSONB`
   - `error_message TEXT`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`
   - `updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`
   - `completed_at TIMESTAMPTZ`
   - Index: `idx_agent_tasks_hotel_status ON agent_tasks(hotel_id, status, created_at)`
   - RLS: enable, SELECT/INSERT/UPDATE for authenticated users WHERE hotel_id matches JWT

Apply the same RLS pattern from Phase 1: `(SELECT auth.jwt()) ->> 'hotel_id')::uuid` for JWT caching per-statement.

Apply `set_updated_at` trigger to `hotel_facts` and `agent_tasks` (reuse the trigger function created in 0001_foundation.sql).

<task type="auto">
  <name>Task 2: Memory read/write helpers</name>
  <files>src/lib/agents/memory.ts</files>
  <action>
Create `src/lib/agents/memory.ts` with the following functions. Use the server Supabase client (`createClient` from `@/lib/supabase/server`). All queries must go through RLS (no service role client).

**Important:** These helpers are called from Route Handlers, not Server Components. The server client from `@/lib/supabase/server` works in Route Handlers because they have access to cookies.

Functions to implement:

1. `loadConversationTurns(conversationId: string)`:
   - SELECT from `conversation_turns` WHERE conversation_id = $1, ORDER BY created_at ASC
   - Limit to last 20 turns to prevent context rot (per research pitfall 3)
   - Return as `Anthropic.Messages.MessageParam[]` format:
     - For `role: 'user'` rows: `{ role: "user", content: row.content }`
     - For `role: 'assistant'` rows: `{ role: "assistant", content: row.content }`
     - For `role: 'tool'` rows: reconstruct as tool_result block using `tool_use_id`
   - Handle the case where tool_use content is stored as JSON string -- parse it back to structured content blocks
   - NOTE: Do NOT import from `@anthropic-ai/sdk` yet (not installed). Use inline type definitions that match the Anthropic MessageParam shape: `{ role: "user" | "assistant"; content: string | ContentBlock[] }`. Add a comment: `// TODO: Replace with Anthropic.Messages.MessageParam after SDK install`

2. `persistTurn(conversationId: string, hotelId: string, role: "user" | "assistant" | "tool", content: string, toolUseId?: string)`:
   - INSERT into `conversation_turns` with the given values
   - Return the inserted row

3. `loadSemanticFacts(hotelId: string)`:
   - SELECT * from `hotel_facts` WHERE hotel_id = $1
   - Format as a readable string: group by category, each fact on a line
   - Return formatted string (this goes into the system prompt)

4. `loadEpisodicHistory(hotelId: string, scope: "full" | "recent_30" | "none")`:
   - If scope is "none", return empty string
   - If scope is "recent_30", SELECT last 30 from `guest_interactions` WHERE hotel_id = $1 ORDER BY created_at DESC LIMIT 30
   - If scope is "full", SELECT all (with a safety cap of 100)
   - Format as readable string: each interaction as "Guest {identifier}: {summary} ({sentiment})"
   - Return formatted string (this goes into the system prompt)

5. `persistEpisodicMemory(hotelId: string, guestIdentifier: string, summary: string, sentiment?: string)`:
   - INSERT into `guest_interactions`
   - Return the inserted row

All functions must be async and handle Supabase errors by throwing with descriptive messages (e.g., `throw new Error("Failed to load conversation turns: " + error.message)`).

Do NOT use a service_role client. All queries go through the standard server client which respects RLS. The Route Handler will have the user's session cookie.
  </action>
