Create the SSE streaming endpoint at `src/app/api/agent/stream/route.ts`.

**Runtime choice: Node.js** (not Edge). Per research recommendation and pitfall 4: Edge runtime breaks `@supabase/ssr`'s cookie-based auth. Use Node.js runtime with extended duration:

```typescript
export const runtime = "nodejs";
export const maxDuration = 60; // 60s on Vercel Pro, sufficient for Claude responses
export const dynamic = "force-dynamic";
```

**Implementation:**

1. Export async `POST(req: Request)` handler
2. **Authenticate**: Use `createClient` from `@/lib/supabase/server`. Call `getUser()` to validate session. If no user, return 401 JSON response.
3. **Get hotel_id**: Extract from JWT claims. The server client has the user's session via cookies. Query `supabase.from('hotels').select('id').single()` -- RLS ensures only the user's hotel is returned. Extract `hotel.id`.
4. **Parse body**: `const { message, conversationId } = await req.json()`. Validate both are non-empty strings. If conversationId is not provided, generate a deterministic one: `${hotelId}_owner_chat` (one persistent conversation per hotel owner, per research recommendation).
5. **Create ReadableStream** with SSE encoding:

```typescript
const encoder = new TextEncoder();
const stream = new ReadableStream({
  start(controller) {
    const send = (data: Record<string, unknown>) => {
      controller.enqueue(encoder.encode(`data: ${JSON.stringify(data)}\n\n`));
    };

    // Heartbeat every 15s to prevent proxy timeout
    const heartbeat = setInterval(() => {
      controller.enqueue(encoder.encode(`event: ping\ndata: keep-alive\n\n`));
    }, 15_000);

    // Abort handling
    req.signal.addEventListener("abort", () => {
      clearInterval(heartbeat);
      controller.close();
    });

    // CRITICAL: Fire-and-forget — do NOT await here (pitfall 1)
    invokeAgent({
      role: AgentRole.FRONT_DESK,
      userMessage: message,
      conversationId: effectiveConversationId,
      hotelId: hotel.id,
      onToken: (token) => send({ type: "token", token }),
    })
      .then(() => send({ type: "done" }))
      .catch((err) => send({ type: "error", message: err instanceof Error ? err.message : "Unknown error" }))
      .finally(() => {
        clearInterval(heartbeat);
        controller.close();
      });
  },
});
```

6. Return the stream as a Response with SSE headers:
```typescript
return new Response(stream, {
  headers: {
    "Content-Type": "text/event-stream; charset=utf-8",
    "Cache-Control": "no-cache, no-transform",
    "Connection": "keep-alive",
    "X-Accel-Buffering": "no",
  },
});
```

**Step 1: Create useChatStream hook** (`src/hooks/useChatStream.ts`):

"use client" hook that manages chat state and SSE consumption.

State:
- `messages: ChatMessage[]` (from `@/lib/agents/types`)
- `isStreaming: boolean`
- `error: string | null`

Functions:
- `loadHistory()`: GET `/api/agent/stream?conversationId=...`, parse JSON response, set messages. Call this on mount.
- `sendMessage(text: string)`:
  1. Add user message to state immediately (optimistic)
  2. Set isStreaming = true
  3. Add empty assistant message (will fill as tokens arrive)
  4. Abort any previous request (useRef for AbortController)
  5. POST to `/api/agent/stream` with `{ message: text, conversationId }`
  6. Read response body as stream using `response.body.getReader()`
  7. Parse SSE lines: split on `\n\n`, filter `data: ` prefix, JSON.parse
  8. For `{ type: "token", token }`: append token to last message's content
  9. For `{ type: "done" }`: set isStreaming = false
  10. For `{ type: "error", message }`: set error, set isStreaming = false
  11. Handle `event: ping` lines (skip them - they're heartbeats)
  12. **Handle chunked SSE data**: The reader may return partial lines. Buffer incomplete lines and only process complete `data: ...\n\n` messages.

- `conversationId`: Use a constant `owner_chat` suffix. The server will prepend hotel_id.

Expose: `{ messages, isStreaming, error, sendMessage }`
