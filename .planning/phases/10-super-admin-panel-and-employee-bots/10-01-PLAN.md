**Migration (0010_admin.sql):**

Create a `delete_vault_secret` SECURITY DEFINER function that deletes a Vault secret by UUID. This is the cleanup function called when bot provisioning fails after a successful Vault insert (prevents orphaned secrets). Pattern matches existing `create_bot_token_secret` and `get_bot_token` in 0009_telegram.sql:

```sql
CREATE OR REPLACE FUNCTION public.delete_vault_secret(p_vault_secret_id UUID)
RETURNS VOID
LANGUAGE plpgsql SECURITY DEFINER SET search_path = public
AS $$ BEGIN DELETE FROM vault.secrets WHERE id = p_vault_secret_id; END; $$;
```

Restrict to service_role only (same REVOKE/GRANT pattern as 0009):
```sql
REVOKE EXECUTE ON FUNCTION public.delete_vault_secret(UUID) FROM PUBLIC, anon, authenticated;
GRANT EXECUTE ON FUNCTION public.delete_vault_secret(UUID) TO service_role;
```

<task type="auto">
  <name>Task 2: Create provisionBots Server Action with Vault + setWebhook + cleanup</name>
  <files>src/lib/admin/provisionBots.ts</files>
  <action>
Export two functions from `src/lib/admin/provisionBots.ts` with `'use server'` directive:

**1. `provisionBotForRole(params)`:**

Parameters: `{ hotelId: string; role: 'front_desk' | 'booking_ai' | 'guest_experience' | 'housekeeping_coordinator'; botToken: string; appUrl: string }`.

Returns: `{ success: true; botUsername: string } | { error: string }`.

Implementation steps (following Pattern 4 from research exactly):

1. **HTTPS validation:** If `appUrl` does not start with `https://`, return error immediately: "App URL must be HTTPS for Telegram webhook registration."
2. **Token validation via getMe:** `fetch(`https://api.telegram.org/bot${botToken}/getMe`)` -- parse JSON, check `ok === true` and `result.username` exists. On failure: return `{ error: 'Invalid bot token -- getMe failed' }`.
3. **Store in Vault:** Call `supabase.rpc('create_bot_token_secret', { p_token: botToken, p_name: `hotel_bot_${hotelId}_${role}_${Date.now()}` })`. On failure: return error. Save `vaultId`.
4. **Generate credentials:** `webhookPathSlug = crypto.randomUUID()`, `webhookSecret = crypto.randomUUID().replace(/-/g, '')` (32 hex chars).
5. **setWebhook:** `POST https://api.telegram.org/bot${botToken}/setWebhook` with body `{ url: `${appUrl}/api/telegram/${webhookPathSlug}`, secret_token: webhookSecret, drop_pending_updates: true, allowed_updates: ['message'] }`. On failure: call `delete_vault_secret` RPC to clean up Vault, then return error.
6. **Upsert hotel_bots:** Use `.upsert({ hotel_id, role, vault_secret_id: vaultId, bot_username, webhook_secret, webhook_path_slug, is_active: true }, { onConflict: 'hotel_id,role' })` -- handles both new provisioning and re-provisioning. On failure: call `delete_vault_secret` RPC to clean up, return error.
7. Return `{ success: true, botUsername }`.

Use `(supabase as unknown as SupabaseClient)` cast for all supabase table/rpc calls.

**2. `provisionAllBots(params)`:**

Parameters: `{ hotelId: string; tokens: Record<string, string>; appUrl: string }` where keys are role strings.

Returns: `{ results: Record<string, { success: true; botUsername: string } | { error: string }> }`.

Implementation: Run `Promise.all` over all entries in `tokens` that have a non-empty value, calling `provisionBotForRole` for each. Collect results keyed by role. Skip roles with empty/undefined token strings.

**Cleanup pattern:** Every failure path after a successful Vault insert must call:
```typescript
await (supabase as unknown as SupabaseClient)
  .rpc('delete_vault_secret', { p_vault_secret_id: vaultId })
  .catch((e: unknown) => console.error('[provisionBots] Vault cleanup failed:', e))
```
