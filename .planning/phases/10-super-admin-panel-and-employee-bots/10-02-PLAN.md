const { data: hotels } = await (supabase as unknown as SupabaseClient)
  .from('hotels')
  .select('id, name, city, country, created_at, onboarding_completed_at, subscriptions(plan_name, status, trial_ends_at)')
  .order('created_at', { ascending: false })
  .limit(100)

2. **Hotel list display:** Render a table (or card list) with columns: Hotel name, City/Country, Created, Status badge. Status badge logic:

**Hotel Detail Page (src/app/(admin)/admin/[hotelId]/page.tsx):**

Server Component that shows hotel info, existing bot status, a provisioning form, and a deep link.

1. **Load hotel data:** Use `createServiceClient()`. Fetch hotel by ID:
```typescript
const { data: hotel } = await (supabase as unknown as SupabaseClient)
  .from('hotels')
  .select('id, name, city, country, created_at, onboarding_completed_at')
  .eq('id', hotelId)
  .single()
```
If not found, call `notFound()` from `next/navigation`.

2. **Load existing bots:** Query hotel_bots for this hotel:
```typescript
const { data: bots } = await (supabase as unknown as SupabaseClient)
  .from('hotel_bots')
  .select('id, role, bot_username, is_active, created_at')
  .eq('hotel_id', hotelId)
```
Display as a status table: Role | Bot Username | Status | Created. Show green check for active bots, grey dash for missing roles.

3. **Bot provisioning form:** Create a client component `BotProvisionForm` (inline or in the same file using `'use client'`). Actually, since this needs interactivity (form state, loading indicators, error display per role), create it as a **separate client component in the same file or as a nested component**.

The form has 4 text input fields, one per role: Front Desk Token, Booking AI Token, Guest Experience Token, Housekeeping Token. Each field shows:
- If a bot already exists for that role: display "Currently: @{bot_username}" above the input
- The input placeholder: "Paste BotFather token to (re)provision"

On submit, call `provisionAllBots` Server Action with `{ hotelId, tokens: { front_desk: tokenValue, ... }, appUrl: process.env.NEXT_PUBLIC_APP_URL }`. **Important:** `NEXT_PUBLIC_APP_URL` is available client-side via the `NEXT_PUBLIC_` prefix.

Display results per role: green checkmark + "@username" on success, red error message on failure. Use `useActionState` (React 19) or `useState` + manual form submission.

After successful provisioning, call `router.refresh()` to update the server-rendered bot status table.

**CRITICAL: Do NOT display the bot token value after save.** The input fields should be cleared after submission. Only show bot_username as confirmation.

4. **Deep link section:** Below the bot provisioning form, display the Setup Wizard deep link:
```
https://t.me/{SETUP_WIZARD_BOT_USERNAME}?start={hotelId}
```

Read `SETUP_WIZARD_BOT_USERNAME` from a prop passed by the Server Component (via `process.env.SETUP_WIZARD_BOT_USERNAME`). If the env var is not set, show a placeholder message: "Set SETUP_WIZARD_BOT_USERNAME env var to enable deep link generation."

If the env var IS set, display the deep link as a copyable text field with a "Copy" button that uses `navigator.clipboard.writeText()`.

5. **Back link:** Add a link back to `/admin` at the top.

Style consistently with the hotel list page -- Tailwind utilities, card borders, muted-foreground labels.
