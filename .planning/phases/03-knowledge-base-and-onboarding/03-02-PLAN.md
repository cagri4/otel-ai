**1. `KnowledgeBaseEditor.tsx` — Main tabbed Client Component:**

'use client' component that receives `{ facts: HotelFact[], rooms: Room[] }` props.

- Use `useSearchParams` to persist active tab in URL (per research Pitfall 6 fix):
  ```typescript
  const searchParams = useSearchParams()
  const router = useRouter()
  const pathname = usePathname()
  const activeTab = searchParams.get('tab') ?? 'policies'
  function handleTabChange(tab: string) {
    const params = new URLSearchParams(searchParams.toString())
    params.set('tab', tab)
    router.push(pathname + '?' + params.toString())
  }
  ```

- Render shadcn Tabs with 5 tabs: policies, faqs, rooms, recommendations, amenities
- Each fact-based tab (policies, faqs, recommendations, amenities) renders `<FactList>` with facts filtered by that category
- Rooms tab renders `<RoomList>` with rooms data
- Each tab has an "Add" button (Plus icon from lucide-react) that opens the appropriate form dialog
- Tab labels: "Policies", "FAQs", "Rooms", "Local Tips", "Amenities"

**2. `FactList.tsx` — Reusable fact list Client Component:**

Props: `{ facts: HotelFact[], category: HotelFactCategory }`

- Filter facts by category
- Render each fact as a card/row with:
  - Fact text
  - Edit button (Pencil icon) — opens FactForm dialog in edit mode
  - Delete button (Trash2 icon) — calls deleteFact(id) Server Action then router.refresh()
- Show empty state message if no facts for this category: "No {category} entries yet. Add your first one."
- Add button at top opens FactForm in create mode (pass category as default)
- Wrap delete in a confirmation (simple window.confirm is fine for MVP)

**3. `FactForm.tsx` — Add/edit fact Dialog Client Component:**

Props: `{ category: HotelFactCategory, fact?: HotelFact, open: boolean, onOpenChange: (open: boolean) => void }`

- If `fact` prop provided, it's edit mode (pre-fill form); otherwise create mode
- Use react-hook-form + zodResolver(factSchema) for validation
- Form fields: hidden category input (pre-set from prop), fact textarea (5-500 chars)
- On submit:
  - Create mode: call addFact(formData) Server Action
  - Edit mode: call updateFact(fact.id, formData) Server Action
  - On success: close dialog, call router.refresh()
- Use shadcn Dialog, Form, Textarea, Button components
- Show validation errors inline

**4. `RoomList.tsx` — Room list Client Component:**

Props: `{ rooms: Room[] }`

- Render each room as a card with:
  - Room name (bold) + room_type badge
  - Bed type, max occupancy, price note if set
  - Description text (truncated if long)
  - Amenities as comma-separated badges
  - Edit button — opens RoomForm in edit mode
  - Delete button — calls deleteRoom(id) + router.refresh()
- Empty state: "No rooms configured yet. Add your first room type."
- Add button opens RoomForm in create mode

**5. `RoomForm.tsx` — Add/edit room Dialog Client Component:**

Props: `{ room?: Room, open: boolean, onOpenChange: (open: boolean) => void }`

- Use react-hook-form + zodResolver(roomSchema)
- Form fields:
  - name (text input, required)
  - room_type (text input, required — e.g. "standard", "deluxe", "suite")
  - bed_type (text input, optional — e.g. "king", "twin", "queen")
  - max_occupancy (number input, optional, 1-20)
  - description (textarea, optional, max 1000)
  - amenities (text input, optional — comma-separated, e.g. "WiFi, minibar, balcony")
  - base_price_note (text input, optional — e.g. "from $120/night")
- On submit:
  - Create: call addRoom(formData)
  - Edit: call updateRoom(room.id, formData)
  - On success: close dialog, router.refresh()
- Use shadcn Dialog, Form, Input, Textarea, Button, Label components

**Styling pattern:** Follow existing shadcn + Tailwind patterns from the settings page. Use Card for list items, space-y for vertical spacing, muted-foreground for secondary text.

**router.refresh() pattern:** After every Server Action success, call `router.refresh()` to trigger Server Component re-render with fresh data (per research Pitfall 2 fix).
