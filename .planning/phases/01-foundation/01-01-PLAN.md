    5. Create `supabase/migrations/0001_foundation.sql` with the COMPLETE schema. Follow the research patterns EXACTLY:

       a. **Hotels table:**
          - id UUID PK with gen_random_uuid()
          - name TEXT NOT NULL
          - address TEXT, city TEXT, country TEXT
          - timezone TEXT NOT NULL DEFAULT 'UTC' (IANA string)
          - contact_email TEXT, contact_phone TEXT
          - created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
          - updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()

       b. **Profiles table:**
          - id UUID PK REFERENCES auth.users(id) ON DELETE CASCADE
          - hotel_id UUID NOT NULL REFERENCES public.hotels(id) ON DELETE CASCADE
          - full_name TEXT
          - role TEXT NOT NULL DEFAULT 'owner' (for future multi-user support)
          - created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()

       c. **Indexes** (critical for RLS performance):
          - idx_profiles_hotel_id ON public.profiles(hotel_id)

       d. **Enable RLS** on both tables.

       e. **RLS Policies** — use `(SELECT auth.jwt())` wrapping pattern for caching:
          - Hotels: SELECT and UPDATE for authenticated, USING id = JWT hotel_id
          - Profiles: SELECT and UPDATE for authenticated, USING auth.uid() = id

       f. **handle_new_user trigger function:**
          - SECURITY DEFINER SET search_path = ''
          - Creates hotel from raw_user_meta_data hotel_name (default 'My Hotel')
          - Creates profile linking user to hotel
          - Updates raw_app_meta_data with hotel_id
          - TRIGGER on auth.users AFTER INSERT

       g. **custom_access_token_hook function:**
          - Reads hotel_id from event claims app_metadata
          - Injects hotel_id as top-level JWT claim
          - GRANT to supabase_auth_admin, REVOKE from public/authenticated/anon

       h. **updated_at auto-update trigger:**
          - Create a generic `set_updated_at()` function
          - Attach BEFORE UPDATE trigger on hotels table

    6. Create `src/types/database.ts`:
       - Export `Hotel` interface matching the hotels table columns (camelCase for TS, snake_case mapping noted in comments)
       - Export `Profile` interface matching profiles table
       - Export `Database` type that wraps these for Supabase client typing (basic structure — full generated types come later)

    IMPORTANT: The SQL migration file is the source of truth. It must be runnable as a single transaction in the Supabase SQL editor. Include comments referencing the pattern source from RESEARCH.md.
