  <files>

  <action>
    1. Create `src/lib/validations/auth.ts`:
       - `signupSchema`: z.object with email (z.string().email()), password (z.string().min(8, 'Password must be at least 8 characters')), hotelName (z.string().min(1, 'Hotel name is required').max(100)), fullName (z.string().min(1).max(100).optional())
       - `loginSchema`: z.object with email (z.string().email()), password (z.string().min(1, 'Password is required'))
       - Export types: SignupInput, LoginInput via z.infer

    2. Create `src/components/forms/signup-form.tsx` ('use client'):
       - Use react-hook-form with zodResolver(signupSchema)
       - Fields: email, password, hotelName, fullName (optional)
       - Use shadcn Form, FormField, FormItem, FormLabel, FormControl, FormMessage, Input, Button components
       - On submit:
         a. Call `supabase.auth.signUp({ email, password, options: { data: { hotel_name: hotelName, full_name: fullName } } })`
         b. Handle error: display error message via form state
         c. On success: call `await supabase.auth.refreshSession()` to force new JWT with hotel_id claim (CRITICAL — see Pitfall 4 in research)
         d. Redirect to `/` (dashboard) using `router.push('/')`
       - Show loading state on submit button
       - Display link to /login for existing users

    3. Create `src/components/forms/login-form.tsx` ('use client'):
       - Use react-hook-form with zodResolver(loginSchema)
       - Fields: email, password
       - Use shadcn Form components
       - On submit:
         a. Call `supabase.auth.signInWithPassword({ email, password })`
         b. Handle error (display message)
         c. On success: redirect to `/` (dashboard) using `router.push('/')`
         d. Use `router.refresh()` after redirect to ensure middleware picks up new session
       - Show loading state
       - Display link to /signup for new users

    CRITICAL: After signUp, you MUST call `supabase.auth.refreshSession()` before redirecting. The initial JWT issued at signup time does NOT contain hotel_id because the trigger runs AFTER token issuance. The refresh forces a new token through the Custom Access Token Hook, which injects hotel_id. Without this, the dashboard will show "no hotel found" on first load.

    Use `createClient` from `@/lib/supabase/client` (browser client) for both forms.
    Use `useRouter` from `next/navigation` for redirects.
  </action>

    - `src/lib/validations/auth.ts` exports signupSchema, loginSchema, SignupInput, LoginInput
    - signup-form.tsx contains `signUp` call with hotel_name in options.data
    - signup-form.tsx contains `refreshSession()` call after successful signup
    - login-form.tsx contains `signInWithPassword` call
    - Both forms use shadcn Form components and zodResolver
  </verify>
