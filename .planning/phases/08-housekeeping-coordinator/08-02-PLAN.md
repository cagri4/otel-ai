```
export async function runHousekeepingQueue(): Promise<{ processed: number; errors: number }> {
```

Steps:

2. Fetch all hotels with `id` and `timezone` columns. Use `(supabase as unknown as SupabaseClient)` cast with `Pick<Hotel, 'id' | 'timezone'>[]` type annotation on loop variable (same TypeScript workaround from STATE.md).
3. For each hotel:
   a. Compute `todayStr` and `tomorrowStr` using TZDate from `@date-fns/tz` and `format` from `date-fns` — same timezone pattern as milestoneDispatch.ts:
      ```typescript
      const now = new TZDate(new Date(), hotel.timezone);
      const todayStr = format(now, 'yyyy-MM-dd');
      const tomorrowStr = format(addDays(now, 1), 'yyyy-MM-dd');
      ```
   b. Query **both** `bookings` and `reservations` tables for checkout/check-in dates matching today/tomorrow. Use UNION ALL logic by querying both tables separately and merging results in JS (PostgRES doesn't support UNION):
      - From `bookings`: select room_id (note: bookings may not have room_id FK — check schema; if not, skip bookings for room-level queue and only use reservations which have room_id FK)
      - From `reservations`: select room_id, check_in_date, check_out_date where status != 'cancelled'
      - Deduplicate by room_id + date to avoid double-counting
   c. Build priority list:
      - Priority 1: rooms with `check_out_date = todayStr` (checkout today — must be cleaned)
      - Priority 2: rooms with `check_in_date = todayStr` AND current status NOT 'clean' (check-in today — needs to be ready)
      - Priority 3: rooms with `check_in_date = tomorrowStr` AND current status NOT 'clean' (check-in tomorrow — advance prep)
   d. For priority 2 and 3, check current status in `room_housekeeping_status` — skip rooms already marked 'clean'.
   e. INSERT into `housekeeping_queue` with `ON CONFLICT (hotel_id, room_id, queue_date) DO NOTHING` for idempotency.
   f. Also: set rooms with checkout today to 'dirty' in `room_housekeeping_status` via upsert (rooms being vacated are dirty by definition), with `updated_by: 'cron'`.
4. Clean up old queue entries: DELETE from `housekeeping_queue` WHERE `queue_date < today - 7 days` (keep 7-day rolling history).
5. Return `{ processed, errors }` counts.

Use `Promise.allSettled` with batching if processing multiple hotels (same pattern as milestoneDispatch.ts).

```typescript
export const runtime = 'nodejs';
export const dynamic = 'force-dynamic';
export const maxDuration = 300;

export async function GET(request: NextRequest) {
  const authHeader = request.headers.get('authorization');
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return new Response('Unauthorized', { status: 401 });
  }
  try {
    const result = await runHousekeepingQueue();
    return Response.json({ ok: true, ...result });
  } catch (error) {
    console.error('[housekeeping-queue] Fatal error:', error);
    return Response.json({ ok: false, error: String(error) }, { status: 200 }); // 200 to prevent retry
  }
}
```
