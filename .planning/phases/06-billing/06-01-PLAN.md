  <action>
Create `supabase/migrations/0006_billing.sql` with:

1. **subscriptions table:**
```sql
CREATE TABLE public.subscriptions (
  id                       UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
  hotel_id                 UUID         NOT NULL UNIQUE REFERENCES public.hotels(id) ON DELETE CASCADE,
  plan_name                TEXT         NOT NULL DEFAULT 'trial'
                                        CHECK (plan_name IN ('trial', 'starter', 'pro', 'enterprise')),
  status                   TEXT         NOT NULL DEFAULT 'trialing'
                                        CHECK (status IN ('trialing', 'active', 'past_due', 'canceled', 'paused')),
  trial_ends_at            TIMESTAMPTZ,
  provider                 TEXT         CHECK (provider IN ('iyzico', 'mollie')),
  provider_customer_id     TEXT,
  provider_subscription_id TEXT,
  created_at               TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  updated_at               TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);
```

2. **Index:** `CREATE INDEX idx_subscriptions_hotel_id ON public.subscriptions(hotel_id);`

3. **RLS:**
   - Enable RLS on subscriptions
   - SELECT policy for authenticated: `hotel_id = ((SELECT auth.jwt()) ->> 'hotel_id')::uuid` (same JWT pattern as all other tables)
   - NO INSERT/UPDATE/DELETE policies for authenticated — only service_role writes

4. **updated_at trigger:** Reuse existing `set_updated_at()` function:
```sql
CREATE TRIGGER set_subscriptions_updated_at
  BEFORE UPDATE ON public.subscriptions
  FOR EACH ROW EXECUTE PROCEDURE public.set_updated_at();
```

5. **Extend seed_hotel_defaults trigger** using `CREATE OR REPLACE FUNCTION public.seed_hotel_defaults()` to also INSERT into subscriptions with `plan_name = 'trial'`, `status = 'trialing'`, `trial_ends_at = NOW() + INTERVAL '14 days'` when a new hotel is created. Keep all existing seed logic (facts, room, agents) intact — only ADD the subscriptions insert at the end.

3. Add `Subscription` interface with all columns matching the migration
4. Add `subscriptions` entry to the `Database.public.Tables` type (same pattern as other tables — Row, Insert, Update, Relationships)

Copy the ENTIRE existing function body from 0005 and append the subscriptions insert — do NOT drop existing inserts.
