Create `/billing` page with Server Component + Client Component split (same pattern as other dashboard pages):

**Server Component: `src/app/(dashboard)/billing/page.tsx`**
- Import and call `getSubscriptionStatus(hotelId)` (need hotel_id from Supabase session — use same pattern as other dashboard pages: `createClient()` + `getUser()` then query hotels)
- Import `getProviderForHotel` from plans.ts
- Load hotel data to get `country` field for provider routing
- Pass subscription status + hotel data + provider to BillingClient
- Handle URL search params for status messages (`?status=success`, `?status=failed`, `?status=pending`)
- Show success/error/pending banners based on search params

**Client Component: `src/app/(dashboard)/billing/BillingClient.tsx`**
- Receives props: `subscriptionInfo` (SubscriptionInfo type), `provider` ('iyzico' | 'mollie'), `hotelCountry` (string), `hotelId` (string)
- Display sections:

  A. **Current Plan Card:**
     - Plan name (displayName from PLAN_LIMITS), status badge (trialing=blue, active=green, past_due=red, canceled=gray)
     - Agent limit: "X of Y agents used" (from subscriptionInfo.maxAgents)
     - If trialing: "X days remaining in free trial" with countdown
     - If trial expired: red banner "Your free trial has ended. Subscribe to continue using AI employees."

  B. **Plan Comparison Grid:**
     - Three cards: Starter, Pro, Enterprise
     - Each shows: displayName, agent count, price (TRY or EUR based on provider)
     - Current plan highlighted
     - Subscribe button (for trial/expired users) or Upgrade/Downgrade button (for active users)
     - Downgrade button only shown if on a higher plan

  C. **Subscribe Action (for trial users):**
     - For iyzico: POST to `/api/billing/iyzico/checkout` with planName + customer data
       - For iyzico, show a form collecting: name, surname, gsmNumber, identityNumber, billing address (city, address, country, zipCode) — these are required by Turkish financial regulation
     - For Mollie: POST to `/api/billing/mollie/checkout` with planName
       - Mollie only needs planName (customer details come from hotel record)
     - On success: redirect to checkout URL (iyzico returns form content; Mollie returns checkoutUrl)

  D. **Upgrade/Downgrade Action (for active users):**
     - For iyzico: POST to `/api/billing/iyzico/upgrade` with newPlanName + upgradePeriod='NOW'
     - For Mollie: POST to `/api/billing/mollie/change-plan` with newPlanName
     - Show loading state during API call
     - On error (e.g., "Disable N agents"): show error message inline
     - On success: refresh the page

Use `useState` for loading states and error messages. Use `fetch()` for API calls (not Server Actions — these are API routes). Style with Tailwind — use the existing card/button patterns from the dashboard (no shadcn card needed, use div with border/rounded/shadow classes).

For iyzico customer data collection: use a modal/dialog or inline form section that appears when "Subscribe" is clicked. The identityNumber field should have a note: "Required by Turkish financial regulation."
