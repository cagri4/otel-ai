  <action>
Install iyzipay: `pnpm add iyzipay @types/iyzipay`

Create `src/lib/billing/iyzico.ts`:

1. **Client singleton:**
```typescript
import Iyzipay from 'iyzipay';

export const iyzipayClient = new Iyzipay({
  apiKey: process.env.IYZIPAY_API_KEY!,
  secretKey: process.env.IYZIPAY_SECRET_KEY!,
  uri: process.env.NODE_ENV === 'production'
    ? 'https://api.iyzipay.com'
    : 'https://sandbox-api.iyzipay.com',
});
```

2. **Plan reference code mapping:**
```typescript
const IYZICO_PLAN_REFS: Record<string, string | undefined> = {
  starter: process.env.IYZICO_PLAN_STARTER_REF,
  pro: process.env.IYZICO_PLAN_PRO_REF,
  enterprise: process.env.IYZICO_PLAN_ENTERPRISE_REF,
};
export function getIyzicoPlanRef(planName: string): string {
  const ref = IYZICO_PLAN_REFS[planName];
  if (!ref) throw new Error(`No iyzico plan ref for: ${planName}`);
  return ref;
}
```

3. **initSubscriptionCheckoutForm:** Accepts params (pricingPlanReferenceCode, callbackUrl, customer with name/surname/email/gsmNumber/identityNumber/billingAddress). Returns Promise wrapping the callback-style iyzipay API. Set `subscriptionInitialStatus: 'ACTIVE'` and `locale: 'tr'`.

4. **upgradeIyzicoSubscription:** Accepts params (subscriptionReferenceCode, newPricingPlanReferenceCode, upgradePeriod: 'NOW' | 'NEXT_PERIOD'). Returns Promise wrapping the iyzipay upgrade API call. NOTE: The iyzipay Node library may not expose a direct `subscriptionUpgrade` method — check the library source after install. If not available, use a raw `fetch()` to iyzico's `/v2/subscription/subscriptions/{ref}/upgrade` endpoint with proper HMAC signing. Document the approach taken in a comment.

5. **validateIyzicoSignature:** Helper function that takes (iyziEventType, subscriptionReferenceCode, orderReferenceCode, customerReferenceCode, providedSignature) and validates using HMAC-SHA256 with `merchantId + secretKey + iyziEventType + subscriptionReferenceCode + orderReferenceCode + customerReferenceCode` concatenation. Uses `crypto.timingSafeEqual` for timing-safe comparison.

All functions use `export` for consumption by route handlers.

- Extract iyziEventType, subscriptionReferenceCode, orderReferenceCode, customerReferenceCode

- Use `createServiceClient()` to update subscriptions:

  - `subscription.cancel` -> status = 'canceled' (if iyzico sends this event)
- Always return 200 (even on DB error — log error, prevent retry storm)
- Add `export const runtime = 'nodejs'` (crypto module requires Node.js runtime)

**2. `src/app/api/billing/iyzico/checkout/route.ts` (POST):**
- Authenticate user via `createClient()` + `getUser()` — return 401 if not authenticated
- Read JSON body: `{ planName: 'starter' | 'pro' | 'enterprise', customer: { name, surname, email, gsmNumber, identityNumber, billingAddress } }`
- Get hotel_id from JWT: `user.app_metadata.hotel_id` or query profiles table
- Get pricing plan reference code via `getIyzicoPlanRef(planName)`
- Call `initSubscriptionCheckoutForm()` with callback URL = `${process.env.NEXT_PUBLIC_APP_URL}/api/billing/iyzico/callback`
- Update subscriptions row: set provider='iyzico', provider_customer_id (from customer params or generated), plan_name to the selected plan (status stays 'trialing' until webhook confirms payment)
- Return the checkout form content (HTML or URL) as JSON response

**3. `src/app/api/billing/iyzico/callback/route.ts` (GET):**
- This is the redirect URL after iyzico form submission
- Parse URL search params for token/status
- If success: redirect to `/billing?status=success`
- If failure: redirect to `/billing?status=failed&error=payment_failed`
- If token expired: redirect to `/billing?status=failed&error=token_expired`

**4. `src/app/api/billing/iyzico/upgrade/route.ts` (POST):**
- Authenticate user via `createClient()` + `getUser()`
- Read JSON body: `{ newPlanName: 'starter' | 'pro' | 'enterprise', upgradePeriod: 'NOW' | 'NEXT_PERIOD' }`
- Get hotel_id from JWT
- Fetch current subscription to get provider_subscription_id
- Validate provider is 'iyzico' — return 400 if not
- If downgrading: check current enabled agent count vs new plan's maxAgents. If exceeds, return 400 with message "Disable N agents before downgrading" (per research recommendation: block downgrade, don't auto-disable)
- Call `upgradeIyzicoSubscription()` with current subscriptionReferenceCode and new plan ref
- Update subscriptions row with new plan_name
- Return 200 with updated subscription info
