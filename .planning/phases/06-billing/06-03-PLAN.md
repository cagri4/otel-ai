  <name>Task 1: Mollie client library and subscription helpers</name>
  <files>src/lib/billing/mollie.ts</files>
  <action>
Install mollie SDK: `pnpm add mollie-api-typescript`

Create `src/lib/billing/mollie.ts`:

1. **Client singleton:**
```typescript
import { Client } from 'mollie-api-typescript';

export const mollieClient = new Client({
  security: { apiKey: process.env.MOLLIE_API_KEY! },
  testmode: process.env.NODE_ENV !== 'production',
});
```

NOTE: If `mollie-api-typescript` has a different import path or initialization pattern than shown in the research, adapt accordingly. Check the actual package exports after install. The package may export `createMollieClient` or similar factory. Read the installed package's README or index.d.ts to confirm.

2. **createMollieCustomer:** Accepts `{ name, email, metadata: { hotelId } }`. Calls `mollieClient.customers.create()`. Returns customer object with ID.

3. **createMollieFirstPayment:** Accepts `{ customerId, planName, redirectUrl, webhookUrl, trialEndsAt }`. Calls `mollieClient.payments.create()` with:
   - `amount: { currency: 'EUR', value: '0.01' }` (minimum for mandate)
   - `customerId`
   - `sequenceType: 'first'`
   - `description: 'OtelAI ${planName} - mandate setup'`
   - `redirectUrl`, `webhookUrl`
   - `metadata: { planName, trialEndsAt }`
   Returns payment object with `_links.checkout.href` URL for customer redirect.

4. **createMollieSubscription:** Accepts `{ customerId, mandateId, planName, amountEur, startDate, webhookUrl }`. Calls `mollieClient.subscriptions.create()` with:
   - `amount: { currency: 'EUR', value: amountEur }` (e.g. "29.00")
   - `interval: '1 month'`
   - `mandateId`, `startDate` (ISO date when trial ends)
   - `description: 'OtelAI ${planName}'`
   - `webhookUrl`
   - `metadata: { planName }`
   Returns subscription object.

5. **changeMolliePlan:** Accepts `{ customerId, currentSubscriptionId, mandateId, newPlanName, newAmountEur }`. Cancels current subscription, creates new subscription starting next month. Uses same mandate (mandate persists across subscriptions for the same customer).

6. **validateMollieSignature:** Helper that takes rawBody and providedSignature (from `X-Mollie-Signature` header), validates HMAC-SHA256 using `MOLLIE_WEBHOOK_SECRET`. Uses `crypto.timingSafeEqual`. NOTE: If Mollie classic webhooks do not include `X-Mollie-Signature` header, skip signature validation and rely on the "always fetch from API" security model instead (document this decision). The webhook handler ALWAYS re-fetches from Mollie API regardless.

<task type="auto">
  <name>Task 2: Mollie webhook handler and billing API routes</name>
  <files>src/app/api/webhooks/mollie/route.ts, src/app/api/billing/mollie/checkout/route.ts, src/app/api/billing/mollie/callback/route.ts, src/app/api/billing/mollie/change-plan/route.ts</files>
  <action>
Create four API route handlers:

**1. `src/app/api/webhooks/mollie/route.ts` (POST):**
- Read body with `await request.text()` FIRST
- Parse as form-encoded: `new URLSearchParams(rawBody)` to extract `id` (NOT JSON.parse — Mollie classic webhooks use application/x-www-form-urlencoded per Pitfall 4 in research)
- If `id` is missing, return 200 (don't error — prevent retry)
- If `MOLLIE_WEBHOOK_SECRET` is set, validate `X-Mollie-Signature` header. If not set, skip validation (classic webhooks may not include it — rely on API fetch security)
- Fetch the payment from Mollie API: `mollieClient.payments.get({ id: paymentId })`
- Use `createServiceClient()` for all DB writes (no user session in webhooks)
- Handle **first payment** (sequenceType === 'first'):
  - If status is 'paid': get mandateId from payment, extract planName and trialEndsAt from metadata
  - List customer mandates to find the valid one
  - Create the Mollie subscription via `createMollieSubscription()` with startDate = trialEndsAt
  - Update subscriptions row: set provider='mollie', provider_customer_id, provider_subscription_id, plan_name
- Handle **recurring payment** (subscriptionId present on payment):
  - If status is 'paid': update subscriptions status to 'active'
  - If status is 'failed' or 'expired': update status to 'past_due'
- Always return 200 (same pattern as Twilio webhook — never cause retry storm)
- Add `export const runtime = 'nodejs'`

**2. `src/app/api/billing/mollie/checkout/route.ts` (POST):**
- Authenticate user via `createClient()` + `getUser()` — return 401 if not authenticated
- Read JSON body: `{ planName: 'starter' | 'pro' | 'enterprise' }`
- Get hotel_id from JWT
- Get hotel data (name, owner email) from hotels + profiles tables
- Get subscription to check trial_ends_at
- Create Mollie customer via `createMollieCustomer({ name: hotelName, email: ownerEmail, metadata: { hotelId } })`
- Create first payment via `createMollieFirstPayment()` with:
  - `redirectUrl: ${NEXT_PUBLIC_APP_URL}/api/billing/mollie/callback`
  - `webhookUrl: ${NEXT_PUBLIC_APP_URL}/api/webhooks/mollie`
  - `trialEndsAt` from subscription row (or calculate 14 days if not set)
- Update subscriptions row: set provider='mollie', provider_customer_id = customer ID
- Return JSON: `{ checkoutUrl: payment._links.checkout.href }` for client-side redirect

**3. `src/app/api/billing/mollie/callback/route.ts` (GET):**
- This is the redirect URL after Mollie payment page
- The actual payment status comes via webhook (not here)
- Redirect to `/billing?status=pending` (payment is being processed)
- The billing page will show "Payment being verified" until webhook confirms

**4. `src/app/api/billing/mollie/change-plan/route.ts` (POST):**
- Authenticate user via `createClient()` + `getUser()`
- Read JSON body: `{ newPlanName: 'starter' | 'pro' | 'enterprise' }`
- Get hotel_id from JWT
- Fetch current subscription — validate provider is 'mollie' (return 400 if not)
- If downgrading: check current enabled agent count vs new plan's maxAgents. If exceeds, return 400 with message "Disable N agents before downgrading to {planName}" (same constraint as iyzico upgrade route)
- Get mandate for customer: list mandates, find valid one
- Call `changeMolliePlan()` with current subscription details and new plan info
- Update subscriptions row with new plan_name and new provider_subscription_id
- Return 200 with updated subscription info

Import `PLAN_PRICES` from plans.ts to get the EUR amount for the new plan (`PLAN_PRICES[newPlanName].eur.toFixed(2)`).
