**2. Create `src/lib/telegram/escapeMarkdownV2.ts`:**

Escapes all 18 MarkdownV2 special characters. The regex must cover: `_`, `*`, `[`, `]`, `(`, `)`, `~`, `` ` ``, `>`, `#`, `+`, `-`, `=`, `|`, `{`, `}`, `.`, `!`

```typescript
/**
 * Escapes all MarkdownV2 special characters in plain text.
 * Must be applied to ALL text that should render as plain text in Telegram.
 *
 * Source: https://core.telegram.org/bots/api#markdownv2-style
 * The 18 characters: _ * [ ] ( ) ~ ` > # + - = | { } . !
 */
export function escapeMarkdownV2(text: string): string {
  return text.replace(/[_*[\]()~`>#+\-=|{}.!]/g, '\\$&');
}
```

**3. Create `src/lib/telegram/sendReply.ts`:**

Sends a message via Telegram Bot API's sendMessage endpoint. Uses MarkdownV2 formatting with a plaintext fallback if the MarkdownV2 send fails (HTTP 400 from Telegram usually means parse error from unescaped characters).

```typescript
import { escapeMarkdownV2 } from './escapeMarkdownV2';

interface SendReplyParams {
  botToken: string;    // Plaintext bot token (decrypted from Vault)
  chatId: number;      // Telegram chat.id
  text: string;        // Raw response text from invokeAgent()
}

export async function sendTelegramReply(params: SendReplyParams): Promise<void> {
  const url = `https://api.telegram.org/bot${params.botToken}/sendMessage`;
  const escaped = escapeMarkdownV2(params.text);

  try {
    const res = await fetch(url, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        chat_id: params.chatId,
        text: escaped,
        parse_mode: 'MarkdownV2',
      }),
    });

    if (!res.ok) {
      const errBody = await res.text();
      console.error('[sendTelegramReply] MarkdownV2 failed:', res.status, errBody);

      // Fallback: send as plain text (no parse_mode) — ensures user always gets a reply
      const fallbackRes = await fetch(url, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          chat_id: params.chatId,
          text: params.text,  // Original unescaped text, no parse_mode
        }),
      });

      if (!fallbackRes.ok) {
        const fallbackErr = await fallbackRes.text();
        console.error('[sendTelegramReply] Plaintext fallback also failed:', fallbackRes.status, fallbackErr);
      }
    }
  } catch (error) {
    console.error('[sendTelegramReply] Network error:', error);
  }
}
```

Key design decisions:
- MarkdownV2 as primary format (per research: current spec, better formatting)
- Plaintext fallback on 400 (prevents silent failures from edge-case escaping issues)
- All errors caught and logged — sendReply never throws (fire-and-forget pattern, same as WhatsApp sendReply)
- botToken parameter is the plaintext token decrypted from Vault (caller handles decryption)

       const response = await invokeAgent({

Key design decisions (document in file header):
- `after()` from `next/server` extends serverless function lifetime after 200 is sent — prevents Telegram retry storms (Pitfall 1 from research)
- Non-streaming invokeAgent (no onToken callback) — same pattern as WhatsApp webhook
- Bot token decrypted inside after() callback, not before — minimize time plaintext token is in memory
- All errors inside after() are caught — handler must never crash the post-response phase
- Rate limited and unknown slug requests return 200 (not 429/404) — prevents Telegram from retrying them
