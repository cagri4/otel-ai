**Main export: `runMorningBriefingDispatch()`**

Returns `{ sent: number; errors: number; skipped: number }`.

**Hotel query:**
- Query `hotels` table: `id, name, timezone, owner_telegram_chat_id`
- Filter: `.not('owner_telegram_chat_id', 'is', null)` (skip hotels without Telegram onboarding)
- Use `createServiceClient()` with the SupabaseClient cast pattern: `(supabase as unknown as SupabaseClient)`

**Per-hotel loop:**
For each hotel:
1. Get `owner_telegram_chat_id` as chatId. Skip if null.
2. Query `hotel_bots` for this hotel: `SELECT role, vault_secret_id, is_active FROM hotel_bots WHERE hotel_id = ? AND is_active = true`
3. If no active bots, skip hotel (increment skipped counter).
4. For each active bot role, build a role-specific briefing text and send it from that bot's token.
5. Rate limit: `await sleep(40)` between EACH send (not between hotels — between individual messages). This means if a hotel has 4 active bots, there are 4 sends with 40ms gaps.

**Bot token resolution:**
Reuse the exact `getFrontDeskBotToken` pattern from `trialNotification.ts` but generalized:

```typescript
async function getBotToken(
  supabase: SupabaseClient,
  hotelId: string,
  role: string,
): Promise<string | null> {
  const { data: bot } = await supabase
    .from('hotel_bots')
    .select('vault_secret_id')
    .eq('hotel_id', hotelId)
    .eq('role', role)
    .eq('is_active', true)
    .maybeSingle();

  if (!bot?.vault_secret_id) return null;

  const { data: tokenData } = await supabase.rpc('get_bot_token', {
    p_vault_secret_id: (bot as { vault_secret_id: string }).vault_secret_id,
  });
  return (tokenData as string) ?? null;
}
```

Copy the exact pattern from `src/app/api/cron/trial-notification/route.ts`:

```typescript
import type { NextRequest } from 'next/server';
import { runMorningBriefingDispatch } from '@/lib/cron/morningBriefing';

export const runtime = 'nodejs';
export const dynamic = 'force-dynamic';
export const maxDuration = 300;

export async function GET(request: NextRequest) {
  const authHeader = request.headers.get('authorization');
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return new Response('Unauthorized', { status: 401 });
  }

  try {
    const result = await runMorningBriefingDispatch();
    return Response.json({ ok: true, ...result });
  } catch (error) {
    const message = error instanceof Error ? error.message : String(error);
    console.error('[cron/morning-briefing] Fatal error:', message);
    return Response.json({ ok: false, error: message }, { status: 200 });
  }
}
```

Key points:
- `maxDuration = 300` — mandatory for batch crons (multiple hotels x multiple bots x 40ms delay each)
- Return 200 even on error — Vercel cron does not retry; errors are logged
- CRON_SECRET bearer auth — consistent with all other cron routes
