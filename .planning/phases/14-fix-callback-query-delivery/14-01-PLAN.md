**Re-provision logic:**
1. Use `createServiceClient()` with the `SupabaseClient` cast to query ALL `hotel_bots` rows (no `is_active` filter — inactive bots should also be updated so reactivation works correctly):
```typescript
import { createServiceClient } from '@/lib/supabase/service';
import type { SupabaseClient } from '@supabase/supabase-js';

const serviceClient = createServiceClient() as unknown as SupabaseClient;
const { data: bots, error } = await serviceClient
  .from('hotel_bots')
  .select('hotel_id, role, vault_secret_id, webhook_path_slug, webhook_secret');
```

2. Iterate sequentially (no `Promise.all` — avoid Telegram rate limits at scale). For each bot:
   a. Decrypt the bot token via `get_bot_token` RPC:
   ```typescript
   const { data: botToken } = await serviceClient.rpc('get_bot_token', {
     p_vault_secret_id: bot.vault_secret_id,
   });
   if (!botToken) { /* log skip, continue */ }
   ```
   b. Build the webhook URL: `${appUrl}/api/telegram/${bot.webhook_path_slug}`
   c. Call `setWebhook` with the CORRECT parameters:
   ```typescript
   await fetch(`https://api.telegram.org/bot${botToken}/setWebhook`, {
     method: 'POST',
     headers: { 'Content-Type': 'application/json' },
     body: JSON.stringify({
       url: webhookUrl,
       secret_token: bot.webhook_secret,  // MUST include — preserves auth header validation
       allowed_updates: ['message', 'callback_query'],
       // NO drop_pending_updates — do NOT discard real pending guest messages

   });
   ```
   d. Track success/failure per bot.
